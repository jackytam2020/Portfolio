JAYUclose

JAYU was my final project in my program at BCIT that I worked on with three other groupmates. This project took around 3 months to complete from September 2019 to end of November 2019

JAYU is an app developed for students aiming to improve productivity. More importantly, it hopes to engage students to work collaboratively as a group. The app has four main sections. A calendar that tracks all deadlines, a homework section that allows students to add todo lists of assignments and subsequent task that needs to be checked off to complete each assignment, a classboard section that allows students to post questions and comments, and a marks page calculate grades for each course.

Development Methods

JAYU was developed using React Native and Postgres SQL.

Learning Outcomes

Worked effectively in a collaborative environment.

Learned and became comfortable with React Native.

Understood how to use UseEffect to execute functions at specified conditions and UseState hooks to change UI.

Connected backend PostgresSQL database to front end to populate data.

Problems Faced

Converting class based functions to functional components.

Looking for online resources was difficult when our group first started developing as most resources uses class based functions.

Getting used to styling on react native since certain css properties are not compatible.

Displaying the right information from the database.

Solution to Problems

Studied and read how class based functions are written to begin with.

Discovered that the read functions were not being executed at the right time which lead to incorrect information being populated.

Configured with UseEffect and other event listener to execute read functions at the right times.

Read documentations on react native website to understand how to style properly and how to use React Native components.

Weather Appclose

This a small project that I worked on over the holidays in December 2019. This is a weather app that displays weather information of every city in the world. Additionally, it will have an animated icon to represent the current weather situation of the selected city. The app pulls two APIs which is google places for the dropdown of all the cities in the world, and the DarkSky weather API.

Learning Outcomes

Due to not learning about API calls and not adding APIs to my school final project, I wanted to learn how to make API calls using javascript as I knew that I have to know this as a junior web developer. I built this app by referring to a tutorial. It taught me step by step how to properly make an API call and how to hide API keys as some APIs require payment after a certain amount of calls.

Problems Faced

I had some minor problems of creating my node server as well as connecting the API properly to my script.

Solution to Problems

A solved my server problems by just making sure all my syntax were correct. I had a some syntax error that caused the server to crash. I also had to reread the documentation of the DarkSky API and the google API to understand why my APIs were not connected properly.

Nominationsclose

Nominations is an app that showcases movies that have won awards from 2010-2018. I made this app with another student who helped with the designs and mockups for our semester end project from February 2019 - April 2019

Development Methods

Nominations was developed with HTML, CSS, and JavaScript

Learning Outcomes

Utilize skills learned from courses such as design, html, css, and javascript.

Solidify knowledge on scripting.

Being comfortable with making functions, passing values into functions as parameters, using if else conditions, calling functions, proper styling, UI changes from Javascript, and manipulating arrays and objects.

CRUD (create, read, update, delete) object items.

Problems Faced

Manipulate objects and object items by CRUD.

Wrong movies were being displayed after filter constraints were inputted by the users.

App did not have a responsive layout prior to the final version.

Had very large JSON objects which made it confusing at times to manage.

Solution to Problems

Made extra functions that helped me display the right movies according to filter constraints.

Used the filter function that made a new array from me according to the conditions that I assign it as such as year won and categories selected.

Wrote another function that pushed and popped items where if the users want to return back to the original screen for new conditions.

Revisited the html structure and css properties to add responsiveness to the app.

MySQL Practiceclose

A small assignment I completed for our Web Development class. This assignment allows users to make posts, delete posts, and like or unlike votes. The front end of the app and the script.js that included ajax functions to populate posts were already provided by our instructor for this assignment. I had to write the php code to handle POST and GET requests. In addition, I wrote the sql queries to select the proper data from the database

Development Methods

Written in php and connected with a database using MySQL

Learning Outcomes

Understand how to handle GET and POST requests while writing php functions.

Create and connect a MySQL database.

Write and implement SQL queries to php files to pull data from MySQL database.

Problems Faced

Since the app had multiple php files, I was confused at first as to where I need to take or pass values from. As a result, there were many occasions where I got errors in the console that said variables were undefined.

Solution to Problems

Once I get a better understanding of the sequence of how I should be handling the data, it became clearer to me which php files I should be working on. Each ajax request should link to the right php file that executes a certain task. I then needed to check if it's a GET or POST request in the specified php file. Each php file is connected to another data.php file which contains all the SQL queries that I should be adding.
